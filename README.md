## 🧠 Philosophy

I believe programming is best learned by **creating things**.
Instead of only reading about systems or frameworks, I prefer to:
- experiment with ideas
- build small prototypes
- recreate concepts from scratch
- explore how things actually work internally

Many of my projects exist purely as **learning experiments** or **technical playgrounds**.
Some might be messy, some unfinished, some overly ambitious – but every repository represents a step in understanding something deeper.

## ⚙️ What I Like Building

You'll find all kinds of things here:

- 🎮 small games and game experiments  
- 🧩 algorithm implementations  
- 🛠 developer tools  
- 🧪 random coding experiments  
- 🖥 graphics / rendering experiments  
- 📦 small libraries or utilities  

A lot of projects start with questions like:

> "What happens if I try to implement this myself?"

or

> "Can I build a minimal version of this?"

## 🚀 How I Work

My workflow is usually very exploration-driven:

1. I get curious about something
2. I start coding immediately
3. I learn while building
4. I refactor (sometimes)
5. I move on to the next idea

Not every project is meant to be production-ready.  
Many are **learning artifacts**, experiments, or technical sketches.

## 📚 Things I'm Interested In
